# 一、注入

# sql注入攻击

sqlmap会用，但是手注是真的硬性要求。

# sql注入的两个标准条件：

1.在注入过程中，有可控的参数（即用户可以手动输入）

2.可插入实现攻击的代码与参数传入过程的拼接。

# 巧妙利用错误回显

将语句之中带入语法错误的内容，如果web服务器开启了==错误回显==，可以在回显的内容中得到数据库版本的信息，从而利用对应的注入语句。

# 关闭错误回显的盲注

在web服务器关闭了错误回显时，用到盲注可以判断是否存在sql注入。

# 原理

构造简单的条件判断语句，判断返回界面的变化。

常见的判断and方法判断：

1=1 1=2

# 报错注入中的floor注入详解：

Payload：

Union select count(*),concat((查询语句),0x26,floor(rand(0)*2))x from information_schema.columns group by x; 


AND (SELECT COUNT(*) FROM information_schema.`TABLES` GROUP BY CONCAT((payload),FLOOR(RAND(0)*2)));// 
通用
AND (SELECT 1 FROM(SELECT COUNT(*),CONCAT((SELECT(SELECT(payload) FROM information_schema.`TABLES` LIMIT 0,1),FLOOR(RAND(0)*2))X FROM information_schema.`TABLES` GROUP BY X)a)
//通用

select 1,2

以1，2作为字段名，打印输出表中的所有信息（若无1，2则直接输出1，2在表中）如图：

count(*) 将所选中表的数据条数显示出来

floor(rand(0)*2))x

floor是取小数点，向下取整，rand()将随机出现一个介于0，1之间的小数，但是rand(0)则根据数据条数的递增，成为一组固定数值，如图：

乘以二取整之后前六位为011011，是一组固定的数字

group by:

group by key 在执行时循环读取数据的每一行，将结果保存于临时表中。读取每一行的key时，如果key存在于临时表中，则更新临时表中的数据（更新数据时，不再计算rand值）；如果该key不存在于临时表中，则在临时表中插入key所在行的数据。（插入数据时，会再计算rand值）

简单的说，就是在执行分组语句时，会建立一个虚拟表，有相同数据直接存入，没有则新建

问题出在当第一次计算的0插入时会再次进行计算rand，得到1插入，再算下一个是1，直接加加，再下一个是0，发现没有0，要插入，但插入时**==再次计算==**发现是1，且已经存在1了，此时有冲突，就会报错。

# 报错注入的其他函数

updatexml函数

报错的原理：

因为在updatexml函数传参中，第二位Xpath路径错误即会报错

具有查询功能 并且会再xpath处查询 你将语法构造错误 然后它就会将他查询的结果已报错的形式显示出来

payload：

and updatexml(1,concat(0x7e,(select (select table_name from information_schema.tables where table_schema= '库名' limit 0,1)),0x7e),1) --%20

and updatexml(1,concat(0x7e,(select (select column_name from information_schema.columns where table_name= '表名' limit 0,1)),0x7e),1)

and updatexml(1,concat(0x7e,(select (select id from 库名.表名 limit 0,1)),0x7e),1)

limit的用法

limit i,n

默认输出从i条==开始==的==前==n条数据，所以在查询表名或字段名不符合想法的，换1，1，这样挨个检索要求找到的信息。

extractvalue()

extractvalue(1,concat())  #用法同上

extractvalue(xml_frag,xpath_expr)

函数接受两个参数，第一个为XML标记内容，也就是查询的内容，第二个为XPATH路径，也就是查询的路径。如果没有匹配内容，不管出于何种原因，只要 xpath_expr有效，并且 xml_frag由正确嵌套和关闭的元素组成 - 返回空字符串。不区分空元素的匹配和无匹配。但是如果XPATH写入错误格式，就会报错，并且返回我们写入的非法内容

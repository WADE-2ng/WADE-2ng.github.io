ENSP路由模拟器指令学习

本篇博客主要是对 ENSP常见命令的学习总结（持续更新）

# 路由器相关配置

# 取消警告及提示信息

[R1]un in en

# 清除配置

<R1>reset saved-configuration

# 1.接口配置

int g0/0/0 #进入接口

ip add 192.168.1.1  # 添加ip

# 2.接口信息查询：

dis int brief #只能查询到接口开放状态

dis ip int brief # 接口绑定IP查询

dis ip routing-table #路由表查询

# 交换机配置

int vlan 1 #创建vlan

ip add ip 24 # ip绑定

dis this #查看当前vlan （也适用于路由）

info-center source ds channel 0 log state off trap state off #取消交换机告警

un in en #取消警告


# dhcp配置

dhcp enable

dhcp select interface   //开启接口的DHCP服务功能，选用接口地址池

# 设置access口
port link-type access

# 静态路由配置：

ip route-static 你要去的ip 要去ip的掩码 必须经过的ip地址

# ospf 默认路由发布

ip route-static 0.0.0.0 0 Loopback 2 #默认路由配置

ospf 1

default-route-advertise #ospf发布

# ospf DR/BDR选举控制

在未设置优先级时，默认优先级都为1。且此时两个路由器会选择route-id较大的那个作为DR，另一个为BDR。

# 正确查看DR和BDR ！！！！
[R1]dis ospf peer (与R1直连的路由器loop ip) #才能找到对应的DR，直接peer会显示邻居DR

	int g0/0/0
	
	ospf dr-priority 200 #修改优先级

[R2]int g0/0/1

	ospf dr-priority 100 
	
	#注意！！！
	#在端口修改优先级之后要重启端口
	# shutdown /  undo shutdown

# vlan结合trunk、access、hybird接口配置

[SW1] vlan batch 10 20 30
	  # access口
	  port link-type access
	  port default vlan 20
	  
	  #trunk口
	  port link-type trunk
	  port trunk allow-pass vlan 10 20 30
	  
	  #hybird口
	  port hybrid untagged vlan 10 30
	  port hybrid pvid vlan 10

# vlan 划分
vlan batch 10 20 （逐个划分）
vlan batch 10-12 （范围划分）

dis port vlan active #交换机各接口接口类型、接口属于哪个vlan（pvid）以及untagged / tagged list回显

# 三种接口的区别以及vlan tagged untagged的解释

ensp交换机默认接口的类型是hybird类型（ 混合接口）在将接口划分至vlan时不可直接由hybird接口划分

# 1.access

一个access口仅仅支持被划分在一个vlan下

access口vlan的划分：

接口模式下修改接口类型、p d v （vlan号）

[Huawei]int e0/0/3
[Huawei-Ethernet0/0/3]port link-type access
[Huawei-Ethernet0/0/3]port default vlan 10
[Huawei-Ethernet0/0/3]dis this
#
interface Ethernet0/0/3
 port link-type access
 port default vlan 10
#
return

此时查看active ：

Eth0/0/3            access       10      U: 10

access接口、pvid为vlan号、untagged list=10

==untagged list==简而言之就是在发送数据包时，如果匹配上untagged list 则删除tag后发送

而==tagged list==就是如果匹配上tagged vlan list ，则保留tag直接发送。即不做任何改变的发送

# 2.trunk

trunk口可被划分至多个vlan下

# access改trunk（已经含有配置项）

1.==交换机下==清除现有所有配置，但是此时接口会关闭 clear configuration int e0/0/3

2.undo shutdown 开启接口

trunk口vlan的划分：

plt设置接口类型、port trunk allow-pass vlan ()设置允通过vlan p t a v 10

[Huawei-Ethernet0/0/3]p l t 
[Huawei-Ethernet0/0/3]port trunk allow-pass vlan 10
[Huawei-Ethernet0/0/3]dis this
#
interface Ethernet0/0/3
 port link-type trunk
 port trunk allow-pass vlan 10
#
return

查看active：

Eth0/0/3            trunk        1       U: 1
                                         T: 10

pvid此时为1，可允许vlan10的包到达，但是由于ping包回复时vid=10，与pvid不相等，不允许发出。故此时ping得不到回复

# 试验

trunk口先行设置pvid=10 而不进行allowed设置

active查看：

Eth0/0/4            trunk        10      T: 1

只是单纯的进行了pvid的赋值，但是没有allow vlan10的进入，数据包根本就进不去4接口

所以==tagged list==也是当接收的数据包打了tag，判断list中是否含有tag，是否无条件放行的一张列表

一个简单的ping包发出，经过接口1，打上接口1对应的pvid。发送至接口2，接口二进行vid与pvid匹配（或tag list中有vid），相同则放行且如果vid在tagged list，去掉tag发走，不在就保留发走。
